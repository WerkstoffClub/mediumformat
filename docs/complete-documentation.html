01 / Overview

Overview & surfaces

Medium Format is an independent record shop in Jakarta and the platform that operates it. The system is strictly black & white — a near-black canvas in the dark theme, white in the light theme, one neutral grey scale between. Geist sets every word; Geist Mono sets the figures that must align (prices, catalogue numbers, barcodes, IDs). Colour appears only to signal status, plus a per-channel key for scannability.

Storefront
Public catalogue

Dark-first commerce: release grid, filters, release detail, cart, Xendit checkout, account, news. Now-Playing bar carries the brand.

Back-office
11-module console

Dense operational tool: dashboard, inventory, locations, orders, customers, vouchers, newsletter, POs, news, channels, preferences.

POS
In-store terminal

Touch point-of-sale: product browser, cart, and a checkout with bank EDC terminals + Xendit gateway + cash.

Character: plain, modern, understated. The only mark with expression is the logomark — a record seen side-on. Everything else stays quiet so the records and artwork are what you look at.

Elevation

Depth comes from surface steps + hairlines, not heavy shadow. Shadow appears only on things that float:

  • 0 Flat — default; surface + hairline
  • 1 Drop — 0 1 2 rgba(0,0,0,.5); dropdowns
  • 2 Float — 0 4 12 rgba(0,0,0,.6); Now-Playing, sticky bars
  • 3 Dialog — 0 8 24 rgba(0,0,0,.7); modals, sheets
  • Focus — 2px ring in the accent colour

  • Back-office sidebar — 11 modules, icon+label, Orders pending dot

Panel / section header bar (v2.1)

The labelled strip atop a content panel/card is a solid accent bar — black bar + white text in light, inverted in dark (it follows --accent/--accent-text). Applies to panel/section headers only — not page titles, table column rows, sidebar/topbar/nav, or dialog headers. Status badges & channel tags inside a header keep their own colour.

06 / Feature

Locations & shelves

Stock is tracked across multiple physical locations, each with its own shelves. Reached from Inventory → Locations.

  • Location management — add / edit / remove locations (Main Store, Warehouse, Pop-up, Consignment), each with type, address, item count, shelf count and low-stock figure.
  • Shelf editor — add/remove shelves per location (A1, A2, C2, Crate 1…).
  • Stock-by-shelf monitor — every item's location + shelf + qty + stock status, filterable by location.
  • Per-location allocation on the release form — the Add/Edit release "Stock & location" section allocates quantity across locations + shelves, with a live total ("N across M locations").
  • Item labels print the item's location · shelf (see §07).
07 / Feature

Barcode labels

Every inventory row can print a physical item label for shelf tracking.

  • Row action Print barcode opens a print-ready label: Medium Format mark, scannable barcode, SKU (mono), artist — title, cat# · condition · price, and the item's location · shelf.
  • The label always renders black-on-white (prints correctly regardless of theme); a print stylesheet isolates just the label.
  • SKU is derived deterministically per release; barcode encodes the SKU.
  • Available on both desktop and the mobile inventory cards.

09 / Feature

POS payments — EDC + Xendit

The POS checkout offers two clearly separated payment families plus cash.

A · Card terminal (EDC)

Physical acquirer terminals: BCA, Mandiri, BNI, BRI, CIMB Niaga, Permata. Selecting a bank runs the terminal flow: insert/tap/swipe → reading → processing → Approved panel with Approval Code, RRN/Ref, Batch, Trace, masked PAN, scheme (Visa/Mastercard/GPN) and entry mode. A Declined path is included.

B · Xendit (online gateway)

"Secured by Xendit": QRIS (scan block), Virtual Account (BCA/Mandiri/BNI), e-wallet (GoPay/OVO/DANA/ShopeePay), and card. Each shows an appropriate waiting/confirmed state.

Cash supports amount-tendered → change due. Completing a sale prints a receipt showing the payment method used (e.g. "EDC · BCA · Visa chip" or "QRIS · Xendit"). The storefront checkout uses Xendit for online orders.

11 / Prototype

Prototype, preview & hub

  • index.html — prototype hub: every screen grouped by surface, Desktop/Mobile toggle, and wired click-through flows (catalogue → release → cart → checkout → confirmation; POS → charge → EDC → receipt; orders → detail → inventory).
  • preview.html — single-file preview: all 22 screens in one place with a Desktop/Mobile toggle, Stage + Gallery views, and open-in-tab.
  • All pages self-contained HTML5; only external dependency is Google Fonts. Pages cross-link so the real flows can be walked before development.
12 / Deployment

Deployment

The static prototype deploys to mediumformat.info on the VPS at vps.lacak.live.

  • deploy/nginx/mediumformat.info.conf — HTTP→HTTPS, HSTS + security headers, CSP, gzip, caching, pretty URLs, 404 handling. A Caddyfile alternative provides automatic HTTPS.
  • deploy/deploy.sh — one-command rsync-over-SSH deploy (config in deploy/deploy.env). deploy/README.md is the full runbook (DNS, certbot, deploy).
  • Web-root assets: favicon.svg (theme-aware), robots.txt, sitemap.xml, site.webmanifest, branded 404.html.
Follow-up for the real product: replace the static pages with the apps/api (NestJS) + apps/backoffice (React) app; move to a nonce-based CSP and self-hosted fonts.
